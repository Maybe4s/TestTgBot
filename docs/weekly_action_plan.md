# Недельный план по текущей цели

Дата старта: 2026-06-14

## 1. СВ-мебель и приглашение в канал

Результат недели: дооформить материалы, смонтировать работу с СВ-мебель и подготовить оформленное приглашение в канал.

Действия:
- Собрать текущие исходники: фото, видео, текст, логотипы, ссылки.
- Определить формат приглашения: картинка, пост, короткое видео или набор из 2-3 вариантов.
- Подготовить текст приглашения: короткий, понятный, с причиной подписаться.
- Смонтировать материал по СВ-мебель.
- Проверить, что готовый вариант подходит для публикации в канал.

## 2. Бриф по колл-центрам

Результат: бриф подготовлен и предфинально проверен.

Действия:
- Использовать `docs/call_center_brief_review.md` как рабочую версию.
- Проверить полноту: цель, аудитория, скрипты, каналы, показатели, отчетность, ответственные.
- Дождаться просмотра от Ярослава или отметить вопросы, которые требуют его решения.
- Подготовить финальную версию после его комментариев.

## 3. Портрет клиентов

Результат после запуска и отпуска: нормальный портрет клиентов, не на ощущениях, а по фактам.

Действия:
- После запуска собрать входящие обращения, диалоги и источники.
- Заполнять портреты по шаблону `docs/client_portrait_framework.md`.
- Разделить клиентов по сегментам.
- Для каждого сегмента описать боли, мотивы, бюджет, сроки, возражения и триггеры покупки.
- Использовать портреты для сайта, объявлений, канала и скриптов.

## 4. Каска и нанесение

Результат: подарок с нанесением, без временной наклейки логотипа.

Действия:
- Не клеить логотип.
- Дать время оперативно закрыть вопрос с нанесением.
- После готовности проверить макет и место нанесения.

## 5. Оформление по дискам

Результат: понятно, что уже есть по дискам и что можно добавить.

Действия:
- Проверить текущие материалы и оформление по чеклисту `docs/drive_audit_checklist.md`.
- Отметить недостающие элементы: обложки, подписи, структура, инструкции, ссылки, единый стиль.
- Составить список улучшений по приоритету.

## 6. Авито

Результат: дождаться информации от Макса и не терять техническую готовность.

Действия:
- Ожидать информацию от Макса.
- Параллельно держать готовым CDP-подключение к уже залогиненному Chrome.
- После информации от Макса выбрать путь: CDP, официальный/партнерский доступ или Albato AI.

## 7. ТЗ по сайту

Результат: согласованное ТЗ и передача в верстку.

Действия:
- Использовать предфинальное ТЗ в `docs/site_tz_draft.md`.
- Использовать передачу исполнителю в `docs/site_handoff_to_layout.md`.
- Согласовать структуру страниц.
- Подтвердить главный CTA и каналы заявок.
- Подготовить список контента для первой версии.
- Передать финальное ТЗ в верстку.

3. Бриф по колл-центрам.
