# План выполнения текущих задач

Дата: 2026-06-14

## 1. Два отснятых объекта

Цель: довести оба объекта до готовых роликов для публикации.

Чеклист:
- Собрать исходники в две отдельные папки.
- Отметить лучшие кадры: общий вид, детали, процесс, результат, люди/эмоция.
- Сделать черновую сборку без цветокоррекции.
- Добавить титры: объект, услуга, ключевой результат, контакт.
- Подготовить версии: 9:16 для Reels/Shorts, 1:1 или 4:5 для ленты, 16:9 для канала/Дзена.

## 2. Улица Московская

Дедлайн ответа: 15 июня.

Текущий пакет: `docs/moscow_video_package_2026-06-15.md`.

Минимальный результат к дедлайну:
- Черновик ролика из остатков.
- Список недостающих кадров, если ролик нельзя собрать чисто.
- Короткий текст для согласования.

Сценарий ролика:
- 0-3 сек: готовый результат или самый сильный кадр.
- 3-10 сек: что было на объекте.
- 10-25 сек: ключевые этапы работ.
- 25-35 сек: детали качества, аккуратность, материалы.
- 35-45 сек: финальный результат и призыв написать/позвонить.

## 3. Instagram

Цель: подготовить профиль к регулярной выкладке и связать его с контент-процессом.

Оформление:
- Аватар с брендом или лицом эксперта.
- Bio: кто, что делает, где, как оставить заявку.
- Ссылка: мессенджер, сайт или мультиссылка.
- Закрепы: работы, отзывы, процесс, цены/расчет, контакты.

Первые публикации:
- 3 ролика по объектам.
- 3 поста-кейса.
- 3 коротких экспертных поста или Reels с советами.

## 4. База контактов

Цель: сформировать базу с телефонами и источниками в нескольких форматах.

Текущий инструмент:
- `maps_parser.py` собирает компании из 2ГИС и сохраняет xlsx/csv/json.
- Города и категории теперь можно задавать при запуске.

Пример запуска:

```bash
python3 maps_parser.py --city "Пенза" --query "ремонт квартир" --query "строительство домов" --prefix "ремонт_стройка_пенза"
```

Поля в выгрузке:
- название, телефоны, мобильные, email, сайт, VK, Telegram, соцсети, рубрика, рейтинг, адрес, поисковый запрос.

Следующий слой:
- Соцсети: отдельно собрать VK/Telegram/Instagram-профили компаний и блогеров.
- Блоги: отдельно собрать авторов/площадки, где есть открытые контакты.
- После сбора разделить базу на сегменты: подрядчики, мебель, ремонт, строительство, блогеры, локальные медиа.

## 5. MAX

Цель: понять рабочий сценарий, а не просто технически отправлять посты.

Что уже есть:
- `MAX:` отправляет текст.
- `MAXPHOTO:` отправляет фото с подписью.
- `PUBLISH:` отправляет фото+подпись во все активные каналы.

Тест:
- Отправить короткий текст.
- Отправить фото с подписью.
- Отправить один пост через мультиканальную публикацию.

Решение после теста:
- MAX как дубль Telegram.
- MAX как отдельный канал с короткими постами.
- MAX как тестовая площадка до появления понятной аудитории.

## 6. Личное видео Дениса

Цель: собрать эмоциональный ролик с соревнований по танцам.

Шаблон разбора исходников: `docs/media_intake_template.md`.

Формат:
- 9:16, 30-60 секунд.
- Ритм под музыку.
- Начало с самого сильного момента.
- В середине: динамика, зал, эмоции, поддержка.
- Финал: лучший кадр, имя/событие/дата.

Нужны:
- Все исходники с соревнований.
- Музыка или направление по музыке.
- Пожелание по стилю: динамично, трогательно, премиально или семейно.
