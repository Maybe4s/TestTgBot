# Авито через залогиненный Chrome: чеклист

## Зачем

Старый способ входа через логин и пароль ловит капчу. Новый способ подключается к обычному Chrome, где владелец уже вошел в Авито вручную.

## Запуск Chrome

1. Полностью закрыть Chrome.
2. Запустить Chrome с debug-портом:

```bash
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome --remote-debugging-port=9222 &
```

3. Открыть Авито в этом Chrome и войти в аккаунт.
4. Проверить порт:

```bash
curl http://127.0.0.1:9222/json/version
```

Если вернулся JSON, Codex/бот сможет подключиться к браузеру.

## Команда боту

```text
AVITO:бани:Баня под ключ от SEDOM:350000:Строим бани и дома. Проект, монтаж, отделка. Гарантия.:/path/photo1.jpg,/path/photo2.jpg
```

## Что изменено в коде

`avito_publisher.py` теперь по умолчанию подключается к `http://127.0.0.1:9222`, проверяет живую авторизацию Авито и открывает отдельную вкладку для публикации.

## Если не работает

- Если порт не отвечает, Chrome был запущен без debug-флага или уже висел старый процесс.
- Если Авито просит логин, нужно войти вручную в этом же Chrome.
- Если интерфейс подачи объявления изменился, нужен свежий скриншот `avito_debug.png`.
- Если CDP не подойдет, запасной путь: Albato AI.
