# Бриф по колл-центрам: предфинальная проверка

Дата: 2026-06-15

## Цель

Подготовить бриф так, чтобы подрядчик по колл-центру понимал задачу, аудиторию, сценарии, отчетность и критерии качества. Ярослав должен посмотреть спорные места перед финальным запуском.

## Что должно быть в брифе

1. Проект и продукт.
2. Цель обзвона:
   - лидогенерация;
   - квалификация;
   - приглашение в канал;
   - запись на консультацию;
   - актуализация базы.
3. Целевая аудитория:
   - сегменты;
   - география;
   - кто принимает решение;
   - кто может быть контактным лицом.
4. База:
   - источник;
   - формат;
   - поля;
   - качество телефонов;
   - правила обработки дублей.
5. Скрипт:
   - открытие;
   - проверка релевантности;
   - короткое предложение;
   - вопросы квалификации;
   - обработка возражений;
   - следующий шаг.
6. Квалификация лида:
   - горячий;
   - теплый;
   - холодный;
   - нецелевой;
   - перезвонить.
7. Отчетность:
   - дата звонка;
   - статус;
   - комментарий;
   - следующий шаг;
   - ответственный;
   - запись разговора, если доступна.
8. Ограничения:
   - что нельзя обещать;
   - какие цены не называть без подтверждения;
   - какие вопросы передавать владельцу.

## Вопросы Ярославу

1. Какой основной результат колл-центра нужен на первой итерации: заявки, встречи, подписки в канал или проверка базы?
2. Какие сегменты обзваниваем первыми?
3. Можно ли использовать текущую базу из 2ГИС как стартовую?
4. Нужна ли запись разговоров?
5. Какие статусы должны быть в отчете?
6. Кто принимает теплые заявки и в какие сроки?
7. Какие формулировки оффера нельзя использовать?

## Минимальный скрипт

Здравствуйте. Меня зовут [имя], я по вопросу [мебели/комплектации/партнерских поставок]. Подскажите, пожалуйста, у вас актуальны закупки или подбор мебели для объектов/клиентов?

Если да:
Можем отправить короткое предложение с моделями, условиями и контактами. Куда удобнее направить: Telegram, WhatsApp или почта?

Если нет:
Понял. Можно уточнить, кто у вас обычно занимается такими вопросами, чтобы не отвлекать не того человека?

## Критерии готовности

- Ярослав посмотрел бриф или явно отметил, что можно запускать без правок.
- Согласованы сегменты первой базы.
- Есть финальный скрипт.
- Есть таблица отчетности.
- Назначен человек, который принимает лиды.
